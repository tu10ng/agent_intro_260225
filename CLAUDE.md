# CLAUDE.md - 项目记忆文件

## 项目概述

这是一个 AI Agent 介绍演示文稿项目，使用 Reveal.js 构建。

- **目标受众**：实在人，关心"怎么用"而非理论
- **演示时长**：45-60分钟
- **幻灯片数量**：28页（第10页为垂直幻灯片组，含3个子页面）

## 核心设计原则

本项目遵循 `how_to_teach.md` 中的教学方法论：

1. **图式化**：先搭框架再填细节，知识形成网络而非孤立点
2. **挂钩点**：新概念必须连接到已知经验（如：上下文窗口 → 会议室白板）
3. **渐进式**：由浅入深，避免认知过载
4. **双轨制**：新手看比喻理解概念，老手看命令直接上手
5. **系统2思维**：强制"过脑子"，而非被动接收

## 关键教学要点

修改演示文稿时务必保持以下核心信息：

- **上下文管理是关键**："塞太多 → 专家变新手"
- **复杂任务必须使用 Plan 模式**：先规划再执行
- **日志是最佳调试语言**：让 AI 看日志而非口头描述
- **永远验证 AI 输出**：信任但要验证

## 方法论应用模式

当需要优化幻灯片内容时，可采用以下模式：

### 垂直幻灯片组模式（适用于核心概念页）

将单页拆分为垂直幻灯片组，包含三个子页面：

1. **挂钩点页**：将新概念映射到前面已讲过的内容（如：功能 → Agent循环）
2. **双轨制页**：用表格展示"新手说法"和"老手用法"
3. **思考页**：添加互动提问，触发系统2思维

**示例**：第10页"Claude Code 能干什么"采用此模式

### 内容精简原则

- 每页聚焦一个核心点
- 表格优于长列表
- 使用 `style="font-size: 0.85em;"` 控制内容密度
- 避免超出页面，宁可拆分子页面

## 修改注意事项

1. **保持双轨制**：每个概念都要有"新手比喻"和"老手命令"两条路径
2. **避免信息过载**：每页幻灯片聚焦一个核心点
3. **比喻要接地气**：使用日常经验（会议室、实习生等）而非抽象概念
4. **Demo 代码保持简单**：目的是演示流程，不是展示复杂功能
5. **中文为主**：面向中文用户，专业术语可保留英文
